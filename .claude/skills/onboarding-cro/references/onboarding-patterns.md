# Patrones Onboarding CRO

## Activacion

La activacion debe ser una accion observable que predice retencion, no solo una pantalla vista.

Ejemplos:

- SaaS gestion: crear primer proyecto y completar una tarea.
- Analytics: conectar fuente y ver primer reporte.
- Marketplace: completar perfil y realizar primera transaccion.
- E-commerce recurrente: guardar preferencias y completar primer pedido.
- Herramienta creativa: crear primer artefacto y exportarlo/compartirlo.

## Primera sesion

- un objetivo principal;
- maximo pasos necesarios para primer valor;
- ayuda cerca del bloqueo;
- progreso visible si hay setup;
- opcion de omitir pasos no criticos;
- datos de ejemplo cuando el estado vacio impide entender valor.

## Empty states

Un buen empty state:

- explica para que sirve el area;
- muestra el resultado esperado;
- ofrece una accion primaria;
- evita texto generico;
- no deja al usuario atrapado.

## Checklist

Usar cuando hay varios pasos necesarios.

- 3 a 7 items.
- orden por valor, no por estructura interna.
- primer item facil.
- progreso visible.
- no bloquear exploracion si no es critico.

## Medicion minima

- signup completed;
- first session started;
- setup step completed;
- activation event;
- time-to-value;
- onboarding completion;
- D1/D7/D30 retention;
- stalled user trigger.
