# Checklist de revision - verificacion de medicion

Antes de entregar:

- [ ] Lei `context.md`, `log.md`, `mensajes.md` y `memory.md` si existen.
- [ ] Identifique cliente, dominio, periodo y servicio afectado.
- [ ] Declare fuentes revisadas y fuentes faltantes.
- [ ] Verifique si cada fuente corresponde al cliente correcto.
- [ ] No pedi ni registre secretos.
- [ ] No modifique produccion ni fuentes vivas.
- [ ] Clasifique Nivel 0/1/2/3.
- [ ] Indique que conclusiones quedan permitidas y cuales no.
- [ ] Deje una sola proxima accion.
- [ ] Si se guardo output, actualice manifest/log segun `protocols/control-artefactos.md`.

Bloquea la entrega final si:

- [ ] Hay contradiccion que cambia la decision.
- [ ] La fuente no puede asociarse al cliente correcto.
- [ ] Se afirma rendimiento, trafico, conversion o ranking sin fuente viva o sin marcar parcial.
- [ ] El siguiente paso implica produccion sin aplicar `protocols/activos-criticos.md`.
