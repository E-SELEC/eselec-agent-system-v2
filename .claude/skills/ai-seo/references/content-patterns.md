# Patrones de contenido para AI SEO

Estos patrones ayudan a que el contenido sea entendible y extraible. No garantizan citas.

## Bloque de definicion

Uso: "que es [tema]".

```markdown
## Que es [tema]

[Tema] es [definicion directa en una frase]. Sirve para [uso principal] y es relevante porque [motivo de negocio o usuario].
```

Reglas:

- responder en la primera frase;
- evitar introducciones largas;
- incluir contexto especifico del cliente si procede.

## Bloque paso a paso

Uso: "como hacer [accion]".

```markdown
## Como [accion]

Para [accion], sigue estos pasos:

1. [Paso claro]
2. [Paso claro]
3. [Paso claro]

Resultado esperado: [resultado].
```

## Tabla comparativa

Uso: "[A] vs [B]", "mejor opcion para [caso]".

```markdown
| Criterio | Opcion A | Opcion B |
|---|---|---|
| Mejor para | ... | ... |
| Coste | ... | ... |
| Ventaja | ... | ... |
| Limite | ... | ... |
```

Reglas:

- ser justo;
- citar datos si hay cifras;
- explicar la decision despues de la tabla.

## Bloque FAQ

Uso: preguntas reales de usuario.

```markdown
### [Pregunta real]

[Respuesta directa]. [Matiz o ejemplo]. [Siguiente accion si aplica].
```

Reglas:

- FAQ visible en pagina si se usara schema;
- no crear preguntas artificiales;
- usar lenguaje natural.

## Bloque de evidencia

Uso: afirmaciones que deben ser creibles.

```markdown
[Afirmacion]. Fuente: [fuente], [fecha]. Implicacion: [que significa para el usuario].
```

Reglas:

- usar fuente primaria cuando sea posible;
- no reciclar estadisticas antiguas sin fecha;
- separar dato, interpretacion y recomendacion.

## Checklist de extractabilidad

- [ ] La respuesta aparece antes de la explicacion larga.
- [ ] Cada seccion responde una pregunta.
- [ ] Los parrafos clave funcionan fuera de contexto.
- [ ] Hay tablas cuando se comparan opciones.
- [ ] Hay fuentes para cifras o claims fuertes.
- [ ] Hay fecha de actualizacion si el tema cambia.
- [ ] Hay autor o responsabilidad editorial cuando importa.
- [ ] La pagina enlaza hacia la conversion o servicio adecuado.
