# Checklist de revision - analytics tracking

Antes de entregar:

- [ ] Defini la decision que la medicion debe informar.
- [ ] Lei contexto, memoria, log, mensajes y tasks si existen.
- [ ] Declare herramientas usadas o pendientes.
- [ ] Separe auditoria, plan, implementacion y validacion.
- [ ] Clasifique Nivel T0/T1/T2/T3.
- [ ] Cada evento tiene trigger, propiedades y decision asociada.
- [ ] No inclui PII ni secretos.
- [ ] Marque conversiones solo para acciones valiosas.
- [ ] Revise riesgo de duplicados.
- [ ] Revise consentimiento/privacy si aplica.
- [ ] No toque produccion sin Orden de Cambio.
- [ ] Indique siguiente accion unica.

Bloquea si:

- [ ] No hay objetivo de negocio o conversion.
- [ ] Se requiere acceso sensible no disponible.
- [ ] Hay datos contradictorios que cambian el plan.
- [ ] Se pide publicar GTM/GA4/pixels sin aprobacion.
- [ ] El plan envia PII a analytics o pixels.
