# Checklist de revision - Content Strategy

Antes de entregar, comprobar:

## Contexto

- [ ] Lei contexto, memoria, log, mensajes y tasks si existen.
- [ ] Revise outputs/manifest por estrategias previas.
- [ ] Declare fuentes y datos faltantes.
- [ ] No repeti contenidos ya fallidos o descartados en log.

## Estrategia

- [ ] Hay objetivo principal y secundario.
- [ ] La audiencia y oferta son claras.
- [ ] Los pilares conectan con servicios reales.
- [ ] Cada tema tiene intencion y etapa.
- [ ] La prioridad usa evidencia, no solo intuicion.
- [ ] El calendario es realista para recursos disponibles.

## Integracion

- [ ] SEO se conecta con `seo-audit` si aplica.
- [ ] AI SEO se conecta con `ai-seo` si aplica.
- [ ] Hubs/URLs se conectan con `site-architecture` si aplica.
- [ ] FAQ/schema se conecta con `schema-markup` si aplica.
- [ ] Publicacion real queda bloqueada por Orden de Cambio.

## Calidad

- [ ] Clasifique nivel CS0-CS3.
- [ ] No invente volumen, dificultad, conversiones ni rankings.
- [ ] Deje una siguiente accion unica.
- [ ] No inclui secretos, PII, exports brutos ni claims no verificables.
