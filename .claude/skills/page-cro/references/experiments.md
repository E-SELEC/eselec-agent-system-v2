# Experimentos CRO

No todo cambio necesita A/B test. Usa esta referencia para proponer tests solo cuando haya medicion y trafico suficiente.

## Hero

| Test | Hipotesis |
|---|---|
| Titular claro vs emocional | claridad aumenta scroll/click |
| CTA especifico vs generico | valor explicito aumenta clicks |
| Hero con prueba vs sin prueba | confianza temprana aumenta conversion |
| Imagen producto/persona vs abstracta | visual concreto reduce incertidumbre |

## Landing

| Test | Hipotesis |
|---|---|
| Quitar navegacion | reduce distraccion en trafico Ads |
| Formulario arriba vs despues de prueba | cambia balance friccion/confianza |
| Pagina corta vs completa | depende de complejidad de oferta |
| Testimonio antes del CTA | reduce duda |

## Servicio/local

| Test | Hipotesis |
|---|---|
| CTA llamada vs formulario | usuarios locales convierten mas por telefono |
| Mostrar zona/localidad arriba | aumenta relevancia local |
| Resenas cerca de CTA | aumenta confianza |
| Servicios en cards vs lista | mejora escaneo |

## Pricing

| Test | Hipotesis |
|---|---|
| Plan recomendado destacado | reduce indecision |
| FAQ de precio cerca | reduce objeciones |
| Mensual/anual | cambia percepcion de valor |

## Blog/recurso

| Test | Hipotesis |
|---|---|
| CTA contextual inline | convierte mejor que CTA final |
| Recurso relacionado | captura leads fríos |
| CTA servicio segun tema | aumenta relevancia |

## Reglas

- No testear si no hay conversion definida.
- No testear si no hay volumen minimo razonable.
- No testear cambios obvios que corrigen errores.
- Registrar hipotesis: "Si cambiamos X, esperamos Y, porque Z".
- Definir metrica primaria y duracion antes del test.
