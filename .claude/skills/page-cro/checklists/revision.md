# Checklist de revision - Page CRO

Antes de entregar, comprobar:

## Contexto

- [ ] Lei contexto, memoria, log, mensajes y tasks si existen.
- [ ] Vi la pagina, captura o estructura suficiente.
- [ ] Identifique audiencia, oferta y accion primaria.
- [ ] Declare fuente de trafico si existe.

## Diagnostico

- [ ] Revise propuesta de valor.
- [ ] Revise CTA y jerarquia.
- [ ] Revise confianza/prueba.
- [ ] Revise objeciones.
- [ ] Revise friccion.
- [ ] Revise mobile cuando hay captura o acceso.
- [ ] Revise medicion/tracking.

## Calidad

- [ ] Clasifique nivel PC0-PC3.
- [ ] Priorice hallazgos por impacto/esfuerzo.
- [ ] No propuse A/B test sin tracking/trafico.
- [ ] Separe quick wins de cambios de alto impacto.
- [ ] No invente datos de conversion.
- [ ] No propuse claims falsos ni dark patterns.

## Riesgo

- [ ] No toque web/CMS/formularios/Ads sin Orden de Cambio.
- [ ] Marque dependencias con analytics, copy, SEO, desarrollo o formularios.
- [ ] Deje una siguiente accion unica.
