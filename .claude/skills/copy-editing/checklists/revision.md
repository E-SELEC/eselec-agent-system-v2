# Checklist de revision - Copy Editing

Antes de entregar, comprobar:

## Contexto

- [ ] Lei contexto, memoria, log, mensajes y tasks si existen.
- [ ] Revise brand/preferencias si aplica.
- [ ] Identifique objetivo, audiencia y accion primaria.
- [ ] Preserve el mensaje central.

## Edicion

- [ ] Mejore claridad.
- [ ] Ajuste tono sin cambiar la voz.
- [ ] Conecte features con beneficios.
- [ ] Revise claims y pruebas.
- [ ] Sustitui lenguaje generico por concreto.
- [ ] Revise CTA y riesgo.

## Seguridad

- [ ] No agregue claims nuevos.
- [ ] No invente cifras, testimonios, precios, garantias ni resultados.
- [ ] Marque claims que necesitan prueba.
- [ ] No contradice contexto, log ni marca.
- [ ] No se publica sin aprobacion si toca web, Ads, email o CMS.

## Cierre

- [ ] Clasifique nivel CE0-CE3.
- [ ] Explique cambios principales.
- [ ] Deje una siguiente accion unica.
