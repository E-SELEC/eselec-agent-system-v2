## Que debe aprender el Docente

El SEO para IA externo busca que una marca, web, producto, servicio o contenido pueda ser entendido, rastreado, citado, seleccionado o usado como fuente por experiencias de busqueda con IA fuera de la propia web.

Incluye entornos como:

```text
Google AI Overviews
Google AI Mode
Microsoft Copilot / Bing AI Search
ChatGPT Search
Perplexity
Claude Search o experiencias conectadas
otros motores conversacionales
```

La regla principal:

```text
SEO para IA externo no es escribir para robots.
Es hacer que la marca sea clara, verificable, citable y consistente.
```

---

## Que no debe confundir el equipo

El Docente debe separar:

```text
SEO:
visibilidad organica tradicional.

AEO:
respuesta directa a preguntas y consultas conversacionales.

GEO:
optimizacion para experiencias generativas de busqueda.

LLMO:
optimizacion para que modelos y asistentes entiendan, citen o usen una entidad.
```

Pero debe insistir en esto:

```text
No existe una capa magica que sustituya al SEO real.
La base sigue siendo tecnica, contenido util, autoridad, entidad, confianza y medicion.
```

---

## Web nueva vs web existente

### Web nueva

La prioridad es preparar la base para que los sistemas puedan descubrir y entender:

```text
arquitectura clara
HTML accesible
robots.txt revisado
sitemap limpio
schema base
entidad de marca clara
paginas comerciales completas
FAQs reales
contenido citable
medicion de referencias IA
```

### Web existente

La prioridad es auditar visibilidad, rastreo y senales externas:

```text
que consultas conversacionales aparecen
que paginas son citables
que bots estan permitidos o bloqueados
que contenido esta indexado
que referencias llegan desde IA
que marca entiende cada sistema
que fuentes externas validan la entidad
que inconsistencias existen
```

---

## Bots y robots.txt

El equipo no debe tocar robots.txt por intuicion.

Debe diferenciar:

```text
bots de rastreo para busqueda
bots de acciones iniciadas por usuario
bots de entrenamiento
bots de preview o recuperacion de contenido
```

Ejemplos que deben conocerse y verificarse con fuentes vivas:

```text
GPTBot
ChatGPT-User
OAI-SearchBot
PerplexityBot
ClaudeBot
Claude-User
Bingbot
Googlebot
```

Regla:

```text
Permitir o bloquear bots de IA es una decision estrategica,
no una accion tecnica aislada.
```

---

## llms.txt

El Docente debe ensenar llms.txt con prudencia.

Criterio:

```text
Puede ayudar a ciertos sistemas o flujos.
No garantiza citacion.
No reemplaza sitemap, schema, HTML, indexacion ni contenido util.
No es una regla universal de Google.
```

Antes de recomendarlo hay que definir:

```text
que sistema lo usa
que contenido se expone
que riesgo existe
quien lo mantiene
como se valida
```

---

## Contenido citable

Un contenido apto para IA externa debe:

```text
responder pronto
usar headings claros
definir entidades
explicar productos y servicios sin ambiguedad
incluir datos verificables
mostrar pruebas, casos, fuentes o referencias
tener FAQs reales
usar schema cuando aplique
estar en HTML rastreable
mantener consistencia con GBP, Merchant Center, redes y directorios
```

Regla:

```text
Si el contenido es debil para humanos,
tambien sera debil para IA.
```

---

## Auditoria de visibilidad IA

Cada revision debe documentar:

```text
experiencia de IA revisada
prompt o consulta usada
respuesta obtenida
marca citada o no citada
URLs citadas
competidores citados
errores detectados
fuentes externas mencionadas
acciones recomendadas
fecha de revision
```

---

## Medicion

GA4 y reportes deben revisar, cuando existan:

```text
referrals desde chatgpt.com
referrals desde perplexity.ai
referrals desde bing.com/copilot si aplica
trafico organico de Bing
consultas conversacionales en GSC
queries long-tail
conversiones por landing citable
```

No todo trafico IA queda visible.

Por eso la medicion se complementa con pruebas manuales documentadas.

---

## Errores que el Docente debe corregir

```text
Pensar que SEO para IA es solo schema.
Pensar que llms.txt resuelve la visibilidad.
Bloquear bots sin entender consecuencias.
Permitir todos los bots sin evaluar riesgo.
Crear contenido generico para "aparecer en ChatGPT".
No revisar Bing cuando se habla de IA externa.
No medir referrals IA.
No documentar prompts y resultados.
No reforzar entidad de marca.
No cruzar IA con SEO local, ecommerce, Merchant Center y GBP.
```

---

## Regla final

```text
SEO para IA externo exige claridad, rastreabilidad, citabilidad,
consistencia externa y medicion.
No es una moda separada del SEO.
Es una extension exigente del mismo sistema.
```
