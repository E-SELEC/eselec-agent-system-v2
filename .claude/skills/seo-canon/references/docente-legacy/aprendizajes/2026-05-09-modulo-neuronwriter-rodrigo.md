## Idea central

NEURONwriter es una herramienta de optimizacion semantica, briefing de
contenidos, redaccion asistida, revision SEO y apoyo para IA/AEO.

No sustituye a SEMrush, Search Console ni GA4.

La jerarquia correcta:

```text
SEMrush -> encuentra oportunidades, keywords, competencia y gaps.
Google Search Console -> valida que ocurre realmente en Google con la web.
Google Analytics 4 -> valida comportamiento, eventos, leads, ventas y negocio.
NEURONwriter -> ayuda a construir, optimizar y mejorar el contenido de cada URL.
```

Regla:

```text
NEURONwriter no decide que paginas crear.
NEURONwriter ayuda a optimizar mejor una pagina que ya tiene una intencion definida.
```

Orden correcto:

```text
Keyword principal
-> intencion
-> tipo de pagina
-> URL
-> objetivo de negocio
-> NEURONwriter
```

Si el equipo mete una keyword mal elegida, una intencion equivocada o una URL
que canibaliza otra, la herramienta puede ayudar a escribir mejor algo que
estrategicamente esta mal.

---

## Donde encaja en el marco maestro SEO

NEURONwriter se conecta con:

1. SEO On-Page: title, meta, H1, H2/H3, terminos semanticos, preguntas,
   estructura y score de optimizacion.
2. SEO de contenidos: briefs, articulos, guias, landings, actualizaciones,
   contenido informacional y contenido transaccional.
3. SEO para IA/AEO/LLMs: claridad, cobertura tematica, estructura, preguntas y
   respuestas, contenido citable y AI Score.
4. Enlazado interno: sugerencias, anchors, paginas destino y aprobacion manual.
5. Produccion editorial: Content Designer, AI templates, Chrome extension,
   WordPress integration y revision de plagio.
6. Mantenimiento de contenidos: reoptimizacion, importacion de contenido
   existente, versiones, mejoras semanticas y actualizacion periodica.

Las integraciones aceleran ejecucion. No sustituyen revision humana.

## Web nueva vs web existente

### Web nueva

En web nueva, NEURONwriter entra despues de keyword research, arquitectura SEO
y mapeo keyword -> URL.

Flujo:

```text
1. Brief de negocio
2. Keyword research en SEMrush / analisis manual
3. Mapeo keyword -> URL
4. Definicion de intencion
5. Crear query en NEURONwriter
6. Analizar SERP y competidores
7. Crear brief de contenido
8. Redactar estructura
9. Optimizar terminos NLP
10. Revisar Content Score
11. Revisar AI Score, si aplica
12. Revisar plagio, si aplica
13. Publicar en WordPress o CMS
14. Medir despues con GSC y GA4
```

Mentalidad:

```text
En web nueva, NEURONwriter ayuda a construir contenido desde cero con base competitiva.
```

### Web existente

En web existente, NEURONwriter entra despues de detectar una URL con oportunidad
en GSC, SEMrush o GA4.

Flujo:

```text
1. Detectar URL con oportunidad
2. Revisar consulta principal en GSC
3. Validar intencion
4. Importar o pegar contenido actual
5. Crear query en NEURONwriter
6. Comparar contra competidores
7. Detectar terminos faltantes
8. Detectar estructura debil
9. Reescribir o ampliar contenido
10. Mejorar headings
11. Añadir FAQs
12. Revisar enlaces internos
13. Actualizar en CMS
14. Medir impacto en GSC y GA4
```

Mentalidad:

```text
En web existente, NEURONwriter no se usa para escribir mas por escribir.
Se usa para mejorar URLs con potencial real.
```

## Que NO debe hacer el equipo

No usar NEURONwriter para:

- decidir toda la estrategia SEO;
- crear paginas nuevas solo porque sugiere terminos;
- perseguir Content Score 100 a costa de naturalidad;
- meter terminos NLP de forma robotica;
- copiar estructura exacta de competidores;
- publicar contenido generado por IA sin revision humana;
- optimizar una URL sin validar intencion;
- mejorar un articulo cuando se necesita una landing comercial;
- crear contenido que canibalice una URL existente.

Regla:

```text
Mas terminos no siempre significa mejor contenido.
```

Mejor un contenido natural y util con score competitivo que un texto forzado
con score perfecto.

Una query es un analisis SEO para una keyword concreta.


Regla:

```text
No se crea una query para cada keyword secundaria.

```

Las keywords secundarias se trabajan dentro del mismo contenido si responden a
la misma intencion.

Error tipico:


## Seleccion y analisis de competidores

NEURONwriter analiza competidores de la SERP para sugerir estructura, terminos,
preguntas y cobertura.

Pero:

```text
No todo resultado en Google es un competidor util.
```

Pueden aparecer:

- directorios;
- marketplaces;
- marcas gigantes;
- articulos informacionales;
- fichas de producto;
- mapas;
- noticias;
- foros;
- PDFs.

Proceso:

```text
1. Revisar SERP real.
2. Preguntar que tipo de paginas aparecen.
3. Validar competidores dentro de NEURONwriter.
4. Descartar resultados que no coinciden con la intencion.
5. Revisar Content Competition.
6. Extraer patrones y huecos.
```

Interpretacion Content Competition:

- baja: oportunidad posible, revisar demanda real;
- media: trabajable con buen contenido y enlaces;
- alta: exige contenido fuerte, autoridad y mejor estrategia.

Entregable:

```text
Keyword:
Tipo de SERP:
Competidores utiles:
Competidores descartados:
Motivo:
Patrones detectados:
Huecos:
Diferenciacion propuesta:
```

Error tipico:

```text
Copiar la estructura del competidor sin mejorarla.
```

## Content Score

Content Score es una señal de cobertura semantica. No es el objetivo final.

Un buen contenido debe:

1. responder intencion;
2. ser claro;
3. ser util;
4. sonar natural;
5. convertir si es comercial;
6. cubrir terminos relevantes;
7. evitar sobreoptimizacion.

Proceso:

```text
1. Redactar primero con estructura logica.
2. Revisar Content Score.
3. Detectar terminos, headings, longitud, meta, H1/H2 y sobreoptimizacion.
4. Incorporar terminos con naturalidad.
5. Evitar score artificial.
```

Regla:

```text
Mejor un 82 natural y util que un 96 robotico.
```

Entregable:

```text
URL:
Keyword:
Content Score inicial:
Content Score final:
Terminos añadidos:
Terminos descartados:
Motivo:
Observaciones de naturalidad:
```

## NLP Terms

Los NLP Terms son pistas sobre el vocabulario natural que deberia aparecer en
un contenido completo sobre un tema.

No son keywords para repetir. Son conceptos que ayudan a cubrir bien la
intencion.

Clasificacion:

- terminos obligatorios: conceptos centrales;
- terminos utiles: amplian el contenido;
- terminos secundarios: se incluyen si encajan;
- terminos descartados: no aplican al negocio o intencion.

Proceso:

```text
1. Clasificar terminos.
2. Añadir terminos por secciones.
3. Revisar naturalidad.
4. Leer en voz alta si hace falta.
```

Error tipico:

```text
Keyword stuffing semantico.
```

Entregable:

```text
Keyword:
Terminos obligatorios:
Terminos utiles:
Terminos descartados:
Seccion donde se incorporan:
Notas:
```

## Brief de contenido

Un brief es una guia para crear o mejorar una URL. No es el contenido final.

Debe decir:

- que pagina se va a crear;
- para que keyword;
- con que intencion;
- que estructura tendra;
- que temas debe cubrir;
- que terminos debe usar;
- que preguntas debe responder;
- que CTA debe incluir;
- que enlaces internos necesita.

Proceso:

```text
1. Abrir analisis.
2. Revisar keyword, competidores y terminos.
3. Definir tipo de contenido.
4. Construir estructura H1/H2/H3.
5. Asignar NLP Terms por seccion.
6. Añadir FAQs.
7. Añadir enlaces internos.
8. Definir CTA y schema sugerido.
```

Entregable:

```text
Meta sugerida:
H1:

```

Error tipico:

```text
Pasar al redactor solo la keyword y decir "haz un articulo SEO".
```

## Content Designer

Content Designer acelera produccion, pero no reemplaza estrategia, edicion ni
criterio.

Usar para:

- borradores de articulos informacionales;
- guias largas;
- FAQs ampliadas;
- contenidos de soporte;
- borradores para editar.

No usar directamente para:

- paginas comerciales sensibles;
- Home;
- paginas de servicio principal;
- paginas legales;
- contenidos medicos, financieros o legales sin revision experta;
- contenidos de marca que requieren tono preciso.

Proceso:

```text
1. Crear brief manual.
2. Usar Content Designer como borrador.
3. Revisar estructura.
4. Corregir informacion.
5. Añadir experiencia real.
6. Añadir diferenciadores de marca.
7. Revisar NLP Terms.
8. Revisar Content Score.
9. Revisar plagio si aplica.
10. Aprobar con editor humano.
```

Error tipico:

```text
Publicar contenido generado sin edicion.
```

## AI Score

AI Score evalua el contenido para IA/AEO en dimensiones como cobertura tematica,
estructura y claridad.

Distincion:

```text
SEO Score -> ayuda a optimizar para busqueda organica tradicional.
AI Score -> ayuda a mejorar claridad, cobertura y estructura para sistemas de IA.
```

Revisar:

- Topic Coverage: cubre bien el tema?
- Structure: esta bien ordenado?
- Clarity: se entiende facil y puede citarse?

Criterios de mejora:

- respuesta principal pronto;
- definiciones concretas;
- H2/H3 logicos;
- listas utiles;
- frases menos genericas;
- marca clara;
- bloques citables.

Error tipico:

```text
Confundir AI Score con escribir para robots.
```

El contenido debe seguir siendo humano, util y comercialmente claro.

## Plagiarism Checker

Usar en:

- contenido generado con IA;
- redactores externos;
- contenidos largos;
- articulos informacionales;
- paginas importantes de marca;
- contenidos sensibles.

Interpretacion:

No toda coincidencia es plagio. Puede haber frases comunes, nombres de

## Auto-Insert Internal Links

Puede acelerar enlazado interno, pero no debe aprobarse automaticamente.

Proceso:

```text
1. Revisar URL actual.
2. Revisar paginas estrategicas que deben recibir enlaces.
3. Ejecutar sugerencias.
4. Revisar anchors.
5. Revisar paginas destino.
6. Aprobar solo enlaces relevantes.
7. Rechazar enlaces forzados.
8. Publicar cambios.
9. Revisar con crawl o GSC.
```

Buen enlace interno:

- tiene sentido para usuario;
- usa anchor descriptivo;
- apunta a URL relacionada;
- refuerza pagina estrategica;
- no rompe lectura;
- no se siente artificial.

Error tipico:

```text
Aprobar todos los enlaces internos por volumen.
```

Enlazado interno no es cantidad. Es distribucion estrategica de contexto y
autoridad.

## Schema Markup

Schema ayuda a buscadores a entender mejor el contenido, pero no se usa por
usar.

Debe corresponder al tipo real de pagina:

- Article / BlogPosting;
- FAQPage;
- Product;
- LocalBusiness;
- Recipe;
- BreadcrumbList.

Regla:

```text
Solo se marca con schema contenido visible y real en la pagina.
```

Proceso:

```text
1. Identificar tipo de pagina.
2. Confirmar contenido visible.
3. Generar schema si aplica.
4. Revisar JSON-LD.
5. Validar con herramienta externa.
6. Implementar en CMS.
7. Revisar en Search Console si aparecen enhancements.
```

Error tipico:

```text
Usar schema para intentar engañar al buscador.
```

## Regla final para relevo

NEURONwriter no se usa para escribir mas.

```text
Se usa para escribir mejor, cubrir mejor la intencion y optimizar con criterio.
```

Cada trabajo debe poder responder:

```text
Que keyword trabaje.
Que intencion resolvi.
Que URL optimice.
Que competidores analice.
Que terminos añadi.
Que terminos descarte.
Que estructura mejore.
Que enlaces internos añadi.
Que score logre.
Que revision humana se hizo.
Que metrica vamos a revisar despues.
```

Si la persona solo dice "subi el score", todavia no entendio la herramienta.

La respuesta profesional es:

```text
Mejore esta URL para esta intencion, cubri estos huecos, reforce estos terminos,
añadi estos enlaces internos, mantuve naturalidad y ahora vamos a medir si suben
impresiones, clics, posicion, engagement y conversiones.
```
