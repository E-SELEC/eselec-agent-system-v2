## Idea central

El analisis SERP manual consiste en abrir Google, buscar una consulta concreta y
estudiar con criterio que tipo de resultados muestra Google, que intencion
domina, que formatos aparecen, que competidores ganan, que contenido se premia
y que tipo de pagina deberiamos crear u optimizar.

La explicacion simple:

```text
El analisis SERP manual no es mirar posiciones.
Es entender que respuesta espera Google para una busqueda concreta.
```

Regla principal:

```text
Antes de crear una URL, optimizar un contenido o decidir una estrategia, hay que
mirar la SERP real.
```

Las herramientas pueden estimar volumen, dificultad o intencion. La SERP real
muestra que ocurre de verdad.

---

## La SERP manda

SERP significa:

```text
Search Engine Results Page
Pagina de resultados del buscador
```

Regla interna:

```text
La SERP manda.
No la herramienta.
```

No se analiza para imitar. Se analiza para entender.

```text
No se copia la SERP.
Se interpreta la SERP.
```

---

## Que NO es analisis SERP

No es:

- buscar una keyword y mirar quien esta primero;
- copiar al competidor;
- asumir que todo top 10 es bueno;
- mirar solo title;
- mirar solo volumen;
- decidir por intuicion;
- crear contenido porque suena bien;
- perseguir keyword aunque la intencion no encaje.

---

## Donde encaja en el sistema

SERP manual conecta con:

1. Keyword research: validar intencion, oportunidad, variaciones y dificultad
   real.
2. Arquitectura SEO: decidir tipo de URL, evitar canibalizacion y mapear keyword
   -> URL.
3. SEO On-Page: title, meta, H1, H2/H3, intencion y profundidad.
4. Contenidos: formato, FAQs, angulo editorial, profundidad y diferenciacion.
5. SEO local: Local Pack, mapas, negocios cercanos, reseñas e intencion
   geografica.
6. Ecommerce: categorias, productos, Shopping, Merchant listings e imagenes.
7. SEO para IA/AEO: AI Overviews, AI Mode, respuestas sinteticas y contenido
   citable.
8. CRO: etapa del embudo, CTA esperado, comparativas y objeciones.

## Web nueva vs web existente

### Web nueva

En web nueva, el analisis SERP se usa antes de crear arquitectura.

Flujo:

```text
1. Definir negocio
2. Definir servicios/productos
3. Buscar keywords iniciales
4. Analizar SERP manualmente
5. Confirmar intencion dominante
6. Decidir tipo de pagina
7. Mapear keyword -> URL
8. Detectar formatos necesarios
9. Disenar arquitectura
10. Crear brief
11. Crear contenido
12. Medir con GSC/GA4
```

Mentalidad:

```text
En web nueva, SERP manual evita crear paginas que Google no quiere mostrar para
esa intencion.
```

### Web existente

En web existente, la SERP se usa para revisar si la URL actual corresponde a la
intencion real.

Flujo:

```text
1. Detectar keyword/URL con oportunidad
2. Revisar GSC
3. Buscar keyword real
4. Analizar SERP
5. Comparar intencion SERP vs URL actual
6. Decidir si optimizar, crear, fusionar o cambiar enfoque
7. Medir despues
```

Mentalidad:

```text
En web existente, SERP manual evita optimizar una URL equivocada.
```

## Variables antes de analizar

Un analisis SERP es una fotografia tomada en un contexto.

Documentar:

- keyword exacta;
- pais;
- ciudad;
- idioma;
- dispositivo;
- fecha;
- navegador/sesion;
- si hay personalizacion;
- si se reviso mobile;
- si hay SERP local;
- si hay anuncios o features.

Regla:

```text
No existe una SERP absolutamente universal.
```

Elementos:

- marketplaces;

Cada elemento revela algo sobre la intencion del usuario.

## Clasificar la SERP por tipo de pagina

Decisiones:

```text
Si la SERP muestra paginas de servicio -> landing de servicio.
Si muestra landings locales -> landing local + GBP.
Si muestra categorias ecommerce -> categoria.
Si muestra fichas de producto -> ficha de producto.
Si muestra guias -> articulo/guia.
Si muestra comparativas -> comparativa.
Si muestra videos -> considerar formato audiovisual.
Si muestra Local Pack -> SEO local.
Si muestra Shopping/productos -> Merchant Center + ecommerce SEO.
```

Regla:

```text
La decision debe salir de la SERP.
```

## SERPs mixtas, dominadas o debiles

SERP mixta:

- varias intenciones conviven;
- puede requerir pagina con secciones mixtas;
- o separar contenido comercial/informacional si hay intenciones claras.

SERP dominada:

- grandes marcas;
- directorios;
- marketplaces;
- medios fuertes;
- formato que no podemos producir.

Accion:

- no atacar directamente;
- buscar long-tail;
- rodear con contenido de soporte;
- reforzar autoridad;
- trabajar local, GBP o Merchant si aplica.

SERP debil:

- resultados pobres;
- contenidos desactualizados;
- baja calidad;
- poca cobertura.

Advertencia:

```text
SERP debil no siempre significa oportunidad de negocio.
```

## SERP y canibalizacion

La SERP ayuda a decidir si una keyword merece URL propia o debe integrarse en
otra.

Si dos keywords tienen SERPs muy parecidas:

```text
Probablemente pueden vivir en la misma URL.
```

Si las SERPs muestran intenciones diferentes:

```text
Pueden necesitar URLs distintas.
```

Regla:

```text
Una URL = una intencion principal.
La SERP ayuda a comprobar esa intencion.
```

## SERP y herramientas

NEURONwriter:

```text
Ayuda a optimizar contenido, pero SERP manual decide el criterio.
```

SEMrush:

```text
Sugiere oportunidad. SERP confirma.
```

GSC:

```text
Muestra queries reales. SERP ayuda a interpretar si la URL correcta responde.
```

GA4:

```text
Muestra si el trafico de esa intencion genera comportamiento o negocio.
```

GBP:

```text
Si la SERP muestra mapas o Local Pack, GBP entra en estrategia.
```

Merchant Center:

```text
Si la SERP muestra productos/Shopping, Merchant Center debe revisarse.
```

Schema:

```text
Si la SERP muestra rich results, revisar que schema puede aplicar.
```

## Errores comunes

1. Mirar solo el primer resultado.
2. Copiar al competidor.
3. No revisar dispositivo/pais.
4. Crear landing cuando la SERP pide guia.
5. Crear articulo cuando la SERP pide categoria.
6. Ignorar Local Pack.
7. Ignorar Shopping/Merchant.
8. Ignorar directorios.
9. No revisar SERP mixta.
10. Usar NeuronWriter sin mirar SERP.
11. Decidir por volumen sin mirar intencion real.
12. Atacar SERP dominada sin estrategia alternativa.

---

## Regla final para relevo

El analisis SERP manual no se hace para ver quien esta arriba.

```text
Se hace para entender que espera Google y que debe construir el equipo.
```

Cada analisis SERP debe responder:

```text
Que keyword analice.
Desde que pais/ciudad/dispositivo.
Que intencion domina.
Que tipo de paginas aparecen.
Que elementos SERP aparecen.
Quienes son competidores reales.
Que formato espera el usuario.
Que gaps existen.
Que tipo de URL necesitamos.
Si hay riesgo de canibalizacion.
Que accion SEO corresponde.
Como se medira despues.
```
