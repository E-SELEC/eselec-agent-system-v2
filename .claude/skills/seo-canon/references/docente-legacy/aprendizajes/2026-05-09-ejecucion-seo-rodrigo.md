## Que estoy aprendiendo

Este aprendizaje no reemplaza la personalidad del Docente SEO. La usa.

La personalidad dice:

```text
claridad -> coherencia -> orden -> ejemplo -> accion
```

Este marco ensena como ejecutar SEO de forma profesional, especialmente para
formar a una persona nueva.

La idea central:

```text
SEO no es hacer cosas sueltas.
SEO es un sistema donde cada accion afecta a otra.
```

Cambiar una URL, eliminar una pagina, tocar un title, modificar un menu o
publicar contenido sin criterio puede ayudar o puede destruir posicionamiento.

---

## 0. Principio base para cualquier persona nueva

El SEO sirve para que una web pueda ser:

```text
encontrada -> entendida -> valorada -> elegida
```

Por buscadores y por usuarios.

La persona que ejecuta SEO debe pensar siempre en este orden:

```text
1. Google puede acceder a la web?
2. Google puede indexarla?
3. Google entiende de que trata cada pagina?
4. Cada pagina responde una intencion concreta?
5. El contenido es util y confiable?
6. La web tiene autoridad?
7. El usuario puede convertir?
8. Estamos midiendo resultados?
```

Regla:

```text
Nunca se empieza por "vamos a escribir blogs".
Eso es SEO debil.
```

## 1. Diferencia entre web nueva y web existente

### Web nueva

En una web nueva, el SEO se ejecuta antes de construir.

Pregunta principal:

```text
Como disenamos una web que nazca con estructura, intencion, contenido, URLs y medicion correctas?
```

Flujo:

```text
Brief de negocio
-> investigacion
-> keyword research
-> arquitectura SEO
-> mapeo keyword-URL
-> wireframes SEO
-> redaccion
-> desarrollo
-> revision tecnica
-> publicacion
-> medicion
```

### Web existente

En una web existente, el SEO se ejecuta sin romper lo que ya funciona.

Pregunta principal:

```text
Que esta pasando ahora, que tiene valor, que esta roto y que podemos mejorar sin destruir trafico, autoridad o conversiones?
```

Flujo:

```text
Auditoria de datos
-> proteccion de activos SEO
-> auditoria tecnica
-> auditoria de arquitectura
-> canibalizacion
-> optimizacion de URLs existentes
-> mejora de contenidos
-> enlazado interno
-> expansion controlada
-> medicion
```

Resumen:

```text
Web nueva: disenar bien antes de publicar.
Web existente: medir, proteger y corregir antes de expandir.
```

## 2. SEO para web nueva: ejecucion paso a paso

### 2.1. Brief estrategico

**Que es**

Entender el negocio antes de tocar SEO.

Sin brief, una persona puede posicionar palabras que no generan dinero.

**Que debe preguntar el ejecutor**

- Que vende la empresa.
- Cuales son los servicios o productos mas rentables.
- Cuales son los servicios o productos prioritarios.
- A que tipo de cliente se quiere atraer.
- En que ciudad, pais o zona vende.
- Que diferencia real tiene frente a competidores.
- Que objeciones suelen tener los clientes.
- Que canales de contacto se usan: telefono, WhatsApp, formulario, ecommerce o reserva.
- Que accion debe hacer el usuario al entrar.
- Que productos o servicios no interesa posicionar.

**Como se ejecuta**

La persona debe crear un documento con:

```text
Negocio:
Cliente objetivo:
Servicios/productos principales:
Servicios/productos secundarios:
Zona geografica:
Diferenciadores:
Competidores conocidos:
Objetivo principal de la web:
Objetivo secundario:
Acciones de conversion:
Prioridad comercial:
```

**Ejemplo**

```text
Negocio: clinica dental en Madrid.
Cliente objetivo: personas que buscan implantes, ortodoncia e higiene dental.
Servicio prioritario: implantes dentales.
Zona: barrio local en Madrid.
Conversion principal: llamada o formulario.
Conversion secundaria: WhatsApp.
```

**Error tipico**

Empezar keyword research sin saber que producto o servicio deja mas margen.

**Entregable**

Un brief claro que permita decidir que paginas debe tener la web.

**Fuentes recomendadas para estudiar despues**

- Google SEO Starter Guide.
- Google Search Essentials.
- Ahrefs Beginner's Guide to SEO.
- Semrush SEO Essentials.
- Google Search ranking systems guide.

### 2.2. Investigacion de mercado y competencia

**Que es**

Analizar quien ya esta posicionando y por que.

No se inventa la estrategia desde cero. Se estudia la SERP.

SERP significa:

```text
Search Engine Results Page = pagina de resultados de Google
```

**Que debe revisar el ejecutor**

Para cada keyword importante:

- que webs aparecen en top 10;
- si aparecen negocios locales;
- si aparecen ecommerce;
- si aparecen blogs;
- si aparecen directorios;
- si aparecen videos;
- si aparecen resultados de Google Maps;
- si aparecen preguntas frecuentes;
- si aparecen AI Overviews o resultados generativos;
- que tipo de contenido premia Google;
- que tan profundas son las paginas;
- que intencion parece tener la busqueda.

**Como se ejecuta**

```text
1. Buscar la keyword.
2. Anotar los 10 primeros resultados.
3. Clasificar cada resultado.
4. Ver si son servicios, categorias, productos, blogs, comparativas o directorios.
5. Revisar title, H1, estructura, contenido, FAQs, enlaces internos y autoridad.
6. Extraer patrones.
```

**Plantilla**

```text
Keyword:
Tipo de intencion:
Top 10 competidores:
Tipo de pagina que posiciona:
Longitud aproximada del contenido:
Elementos comunes:
Preguntas frecuentes:
Diferenciadores:
Huecos detectados:
Oportunidad:
```

**Ejemplo**

Para "supermercado italiano Madrid":

```text
Intencion: local/comercial.
Google puede mostrar tiendas, Maps, directorios y webs locales.
La pagina ideal combina ubicacion, productos italianos, marcas, entrega,
tienda fisica, confianza y resenas.
```

**Error tipico**

Crear una pagina comercial cuando Google premia guias informacionales, o crear
un blog cuando Google premia paginas de servicio.

**Entregable**

Tabla de competencia por keyword y conclusion:

```text
Para esta keyword debemos crear:
- pagina de servicio
- categoria
- producto
- articulo
- landing local
- comparativa
```

### 2.3. Keyword research

**Que es**

Encontrar las busquedas que hacen los usuarios y decidir cuales merecen una
pagina, un articulo, una categoria o ninguna accion.

Keyword research no es buscar palabras con volumen. Es entender demanda,
intencion y valor comercial.

**Tipos de keywords**

```text
Transaccionales: comprar pasta italiana online
Comerciales: mejor supermercado italiano en Madrid
Locales: supermercado italiano en barrio local
Informacionales: que es el guanciale
Comparativas: guanciale vs panceta
Marca: nombre de marca del cliente
```

**Como se ejecuta**

```text
1. Sacar palabras semilla.
2. Buscar variaciones en herramientas.
3. Revisar Google Suggest.
4. Revisar People Also Ask.
5. Revisar Search Console si la web ya existe.
6. Revisar competidores.
7. Agrupar keywords por intencion.
8. Priorizar por negocio.
9. Decidir que URL debe atacar cada intencion.
```

**Como priorizar**

No se prioriza solo por volumen.

```text
Prioridad = relevancia comercial + intencion + dificultad + oportunidad + margen
```

Una keyword con 70 busquedas puede valer mas que una con 2.000 si atrae
compradores listos para pagar.

**Entregable**

```text
Keyword principal:
Keywords secundarias:
Intencion:
Tipo de pagina:
URL sugerida:
Prioridad:
Observaciones:
```

**Error tipico**

Crear una URL para cada keyword parecida.

---

### 2.4. Mapeo keyword -> URL

**Que es**

Decidir que pagina va a trabajar cada intencion.

Esta fase evita canibalizacion.

**Regla principal**

```text
Una intencion principal = una URL principal.
Una URL principal = una intencion principal.
```

**Como se ejecuta**

```text
1. Tomar la lista de keywords.
2. Agrupar keywords parecidas.
3. Identificar si tienen la misma intencion.
4. Asignar una URL principal por grupo.
5. Definir keywords secundarias.
6. Marcar futuras URLs complementarias.
7. Revisar si la keyword debe ir en blog, landing, categoria o producto.
```

**Ejemplo bueno**

```text
URL: /supermercado-italiano-madrid/
Keyword principal: supermercado italiano Madrid
Secundarias: tienda italiana Madrid, mercado italiano Madrid, productos italianos Madrid
Intencion: encontrar un supermercado italiano en Madrid para visitar o comprar.
```

**Error tipico**

Asignar la misma keyword principal a varias URLs.

### 2.5. Arquitectura SEO

**Que es**

Organizar la web para que usuarios y Google entiendan que es importante.

No es solo menu. Es jerarquia.

**Estructura base**

```text
Home
|
|-- Servicios / Categorias principales
|   |-- Subservicios / Subcategorias
|   |-- Productos / Casos / Variantes
|
|-- Blog / Recursos
|   |-- Guias
|   |-- Comparativas
|   |-- FAQs
|
|-- Contacto / Sobre nosotros / Legales
```

**En web nueva**

Definir paginas principales, secundarias, contenidos de apoyo, menu, footer,
enlaces internos desde Home, breadcrumbs, profundidad y estructura de URLs.

**En web existente**

Hacer crawl, exportar URLs, clasificarlas, revisar trafico, backlinks, paginas
huerfanas, profundidad y reorganizar sin cambiar URLs salvo necesidad real.

**Error tipico**

Crear una arquitectura bonita para diseno, pero inutil para SEO.

## 3. SEO para web existente: ejecucion paso a paso

### 3.1. Auditoria de datos reales

**Que es**

Revisar que esta pasando antes de tocar la web.

En web existente, Search Console es fuente central porque muestra consultas,
paginas, clics, impresiones, posicion y CTR.

**Que revisar**

En Search Console:

- clics organicos;
- impresiones;
- CTR;
- posicion media;
- consultas principales;
- paginas principales;
- paises;
- dispositivos;
- caidas;
- URLs indexadas;
- URLs excluidas;
- errores de cobertura.

En GA4:

- trafico organico;
- eventos;
- conversiones;
- formularios;
- clics en telefono;
- clics en WhatsApp;
- compras;
- ingresos si ecommerce.

**Que buscar**

```text
Muchas impresiones + bajo CTR -> mejorar title/meta.
Posicion 4-15 -> mejorar contenido, enlaces internos y autoridad.
Trafico alto + cero conversiones -> revisar CRO.
Pagina importante sin impresiones -> revisar intencion, indexacion, autoridad o contenido.
Pagina indexada sin valor -> mantener, noindexar, fusionar o eliminar con criterio.
```

**Error tipico**

Auditar visualmente sin mirar datos.

---

### 3.2. Proteccion de activos SEO

**Que es**

Identificar que no se debe romper.

Una web puede tener paginas feas, viejas o mal disenadas que generan trafico,
enlaces o leads. No se eliminan sin analisis.

**Que debe protegerse**

- URLs con trafico;
- URLs con backlinks;
- URLs con conversiones;
- URLs con buenas posiciones;
- URLs enlazadas desde otras webs;
- URLs con enlaces internos importantes;
- titles con buen CTR;
- contenidos que generan leads;
- URLs antiguas con autoridad.

**Clasificacion**

```text
Alta proteccion: trafico, backlinks o conversiones.
Media proteccion: impresiones o enlaces internos.
Baja proteccion: sin trafico, sin backlinks, sin conversion y sin valor.
```

**Error tipico**

Redisenar una web y cambiar URLs sin redirecciones.

---

### 3.3. Auditoria tecnica correctiva

**Que es**

Revisar si hay problemas que impiden que Google rastree, entienda o indexe la
web.

**Que se revisa**

```text
Rastreo: robots.txt, enlaces internos, sitemap, paginas huerfanas.
Indexacion: noindex, canonicals, duplicados, paginas excluidas.
Errores: 404, 500, redirecciones rotas, cadenas.
Rendimiento: velocidad, Core Web Vitals, imagenes, scripts.
Mobile: responsive, botones, contenido visible, formularios.
Schema: valido, coherente, sin marcado enganoso.
```

**Prioridad**

```text
Critico: noindex accidental, robots mal configurado, errores 500.
Alto: 404 importantes, canonicals incorrectos, redirecciones rotas.
Medio: imagenes pesadas, schema incompleto, titulos duplicados.
Bajo: ajustes sin impacto directo.
```

**Error tipico**

Entregar al desarrollador una lista de errores sin prioridad ni solucion.

---

### 3.4. Auditoria de canibalizacion

**Que es**

Detectar si dos o mas URLs compiten por la misma intencion.

Canibalizacion no significa que dos paginas mencionen el mismo tema. Significa
que Google no sabe cual debe posicionar porque ambas responden lo mismo.

**Como se detecta**

- exportar consultas desde Search Console;
- revisar que URLs aparecen para la misma query;
- buscar con `site:dominio.com keyword`;
- revisar titles y H1 similares;
- revisar paginas que alternan posiciones;
- revisar articulos que compiten con servicios.

**Como se corrige**

```text
1. Fusionar y redirigir URLs debiles.
2. Diferenciar intencion.
3. Cambiar enfoque keyword.
4. Mejorar enlazado interno hacia la URL principal.
```

**Error tipico**

Eliminar paginas canibalizadas sin revisar trafico, backlinks o leads.

## 4. SEO On-Page: como ejecutarlo

**Que es**

Optimizar una URL especifica para que responda mejor a una busqueda concreta.

No es solo title, meta y H1.

Incluye:

```text
intencion, keyword, estructura, contenido, semantica, multimedia, enlaces internos,
confianza, CTA y schema.
```

### 4.1. Revision de intencion

Antes de tocar la pagina:

```text
Que quiere el usuario?
Comprar, comparar, aprender, encontrar negocio cercano, pedir presupuesto o resolver duda?
```

Luego se busca la keyword en Google y se ve que tipo de paginas posicionan.

### 4.2. Title SEO

Debe incluir:

```text
keyword principal + diferenciador + ubicacion/marca si aplica
```

Error:

```text
SEO Madrid | SEO Madrid barato | Agencia SEO Madrid
```

### 4.3. Meta description

Debe responder:

```text
que ofreces + para quien + beneficio + CTA
```

No debe copiarse igual en todas las paginas.

### 4.4. H1

Un H1 principal por pagina.
Debe dejar claro el tema central.

Error: usar "Inicio", "Bienvenido" o "Servicios".

### 4.5. H2 y H3

Ordenan la pagina.
Los H2 responden grandes bloques y los H3 desarrollan detalles.

### 4.6. Contenido principal

Debe explicar:

```text
que se ofrece, para quien, donde, que incluye, por que elegirte,
como funciona, que diferencia tienes y que debe hacer el usuario.
```

### 4.7. Imagenes y ALT

Comprimir imagen, nombre descriptivo, ALT natural, imagen real cuando sea posible.

No hacer keyword stuffing.

### 4.8. Enlazado interno

Revisar:

```text
que paginas enlazan hacia esta URL,
a que paginas enlaza esta URL,
que anchor se usa,
si ayuda al usuario,
si refuerza paginas importantes.
```

### 4.9. FAQs

Deben salir de dudas reales, People Also Ask, Search Console, objeciones
comerciales, ventas y soporte.

### 4.10. Schema

Datos estructurados para ayudar a Google a entender el contenido.

Debe ser coherente con contenido visible. No garantiza rich results.

## 5. SEO de contenidos

**Que es**

Crear, mejorar, fusionar o eliminar contenido para cubrir demanda de busqueda y
construir autoridad tematica.

No es publicar por publicar.

**Tipos**

```text
Transaccional: servicios, categorias, productos.
Informacional: guias, tutoriales, explicaciones.
Comparativo: A vs B, alternativas, mejores opciones.
Local: ciudad, zona, barrio.
Autoridad: estudios, casos, recursos propios.
Soporte: FAQs, glosarios, documentacion.
```

**Cuándo crear contenido nuevo**

Solo cuando hay intencion no cubierta, oportunidad clara, apoyo a pagina
comercial, autoridad tematica, duda real o trafico cualificado.

**Cuándo mejorar**

Cuando tiene impresiones sin clics, posicion 4-15, contenido pobre,
desactualizacion o potencial comercial.

**Cuándo fusionar**

Cuando varias URLs atacan la misma intencion o ninguna tiene fuerza sola.

**Cuándo eliminar o noindexar**

Cuando no tiene trafico, backlinks, conversiones, valor, encaje ni utilidad.

Antes se revisa si hay que redirigir.

**Error tipico**

Crear 20 articulos nuevos cuando habia que mejorar 5 paginas con potencial.

## 6. SEO tecnico para principiantes

**Explicacion sencilla**

SEO tecnico es asegurarse de que Google pueda entrar en la web, recorrerla,
leerla, entenderla e indexarla sin problemas.

Si una pagina no puede rastrearse o indexarse, no puede posicionar.

**Proceso**

```text
1. Revisar rastreo.
2. Revisar indexacion.
3. Revisar errores.
4. Revisar velocidad y Core Web Vitals.
5. Revisar mobile.
6. Revisar schema.
```

**Entregable tecnico**

```text
Problema:
Ejemplo de URL:
Impacto:
Solucion:
Prioridad:
Responsable:
```

**Error tipico**

Decir "la web esta lenta" sin especificar que elemento la ralentiza.

## 7. SEO local

**Que es**

Optimizar presencia para busquedas con intencion geografica, especialmente en
Google Maps y resultados locales.

**NAP**

```text
Name, Address, Phone = nombre, direccion y telefono.
```

**Trabajo principal**

```text
Google Business Profile:
nombre, categorias, direccion, telefono, horarios, web, servicios, productos,
descripcion, fotos, resenas, publicaciones, preguntas y respuestas.

Web local:
landing local, NAP visible, mapa, zonas, semantica local, testimonios,
enlaces a GBP y schema LocalBusiness.

Resenas:
cantidad, frecuencia, calidad, palabras usadas, respuesta y sentimiento.

Citaciones:
nombre, direccion, telefono, web y categoria coherentes.
```

**Error tipico**

Pensar que SEO local es solo meter keywords con ciudad en la web.

## 8. SEO Off-Page y autoridad

**Que es**

Todo lo que ocurre fuera de la web y mejora confianza, autoridad, reputacion y
visibilidad.

No es solo conseguir backlinks.

Incluye backlinks, menciones, PR digital, directorios relevantes,
colaboraciones, resenas, presencia sectorial y autoridad de marca.

**Proceso**

```text
1. Auditar backlinks actuales.
2. Identificar oportunidades.
3. Crear activos enlazables.
4. Hacer outreach.
5. Evitar spam.
```

**Error tipico**

Comprar enlaces baratos y creer que eso es autoridad.

## 9. SEO para IA / LLMs / AEO

**Que es**

Preparar marca y contenido para ser entendidos, citados o recomendados por
sistemas de IA, buscadores generativos y respuestas conversacionales.

No reemplaza el SEO tradicional. Lo exige mejor hecho.

**Que se trabaja**

```text
Claridad de marca:
quien eres, que vendes, a quien ayudas, donde trabajas, por que elegirte,
que pruebas tienes.

Entidades:
empresa, persona, servicio, producto, marca, ubicacion, categoria, especialidad.

Contenido citable:
definiciones claras, respuestas directas, listas utiles, comparativas,
datos verificables, FAQs reales, fragmentos autosuficientes.

Presencia externa:
GBP, directorios, redes, medios, resenas, partners, menciones y foros relevantes.
```

**Error tipico**

Pensar que SEO para IA es crear un archivo raro o meter mas schema.

## 10. CRO aplicado al SEO

**Que es**

CRO significa Conversion Rate Optimization.

En espanol:

```text
optimizacion de conversion
```

SEO trae trafico. CRO convierte ese trafico en llamadas, formularios, ventas,
reservas o WhatsApps.

**Que revisar**

```text
Primer pantallazo: se entiende que ofrece la empresa?
CTA: hay boton claro?
Confianza: hay resenas, casos, garantias, fotos reales?
Contacto: telefono clicable, WhatsApp, formulario corto?
Movil: se puede usar facil?
Contenido: resuelve objeciones?
Velocidad: carga rapido?
Medicion: sabemos que convierte?
```

**Eventos GA4 ejemplo**

```text
click_whatsapp
click_phone
form_submit
purchase
booking_start
file_download
```

**Error tipico**

Celebrar que sube el trafico sin saber si suben los leads.

## 11. Medicion y mantenimiento SEO

SEO no termina cuando publicas.

**Semanal**

```text
errores graves, indexacion, caidas, cambios bruscos, formularios, eventos.
```

**Mensual**

```text
clics, impresiones, CTR, posicion media, paginas que suben, paginas que bajan,
keywords nuevas, keywords perdidas, conversiones, leads y ventas.
```

**Trimestral**

```text
arquitectura, contenido antiguo, canibalizacion, competencia, backlinks,
autoridad, SEO local, SEO para IA y roadmap.
```

**Decisiones**

```text
Muchas impresiones + bajo CTR -> mejorar title/meta.
Posicion 4-15 -> mejorar contenido, enlaces internos y autoridad.
Trafico sin leads -> mejorar CRO.
Paginas sin impresiones -> revisar indexacion, intencion o contenido.
Caida fuerte -> revisar cambios tecnicos, updates, competencia o indexacion.
```

**Error tipico**

Hacer reportes llenos de metricas sin decisiones.

```text
20. Crear contenido nuevo solo donde falte
21. Medir impacto
22. Iterar
```

**Que no se debe hacer**

```text
No cambiar URLs sin revisar datos.
No eliminar paginas sin revisar trafico/backlinks.
No tocar titles que funcionan sin motivo.
No redisenar sin proteger conversiones.
No cambiar arquitectura sin redirecciones.
No fusionar contenido sin plan.
No publicar contenido nuevo si hay canibalizacion pendiente.
```

---

## 14. Matriz de prioridad para formar al equipo

Cada tarea debe clasificarse por:

```text
Impacto: Alto / Medio / Bajo
Esfuerzo: Alto / Medio / Bajo
Riesgo: Alto / Medio / Bajo
Tipo: Tecnico / Contenido / On-page / Local / Off-page / CRO / Medicion
```

**Prioridad maxima**

Alta importancia + bajo esfuerzo + bajo riesgo.

Ejemplos:

```text
Mejorar title de pagina con muchas impresiones y bajo CTR.
Anadir enlaces internos hacia URL en posicion 8.
Corregir noindex accidental.
Arreglar CTA roto en pagina con trafico.
```

**Prioridad peligrosa**

Alto impacto + alto riesgo.

Ejemplos:

```text
Cambiar URLs.
Migrar dominio.
Eliminar paginas.
Redisenar estructura.
Fusionar contenidos.
```

Estas tareas requieren revision senior.

## 15. Reglas de oro para la generacion de relevo

```text
No se hace SEO por intuicion.
Se hace SEO con intencion, datos, estructura, ejecucion y medicion.
```

Y:

```text
Cada URL debe tener una razon para existir.
Si no atrae, no ayuda, no convierte, no enlaza, no posiciona y no aporta
confianza, hay que mejorarla, fusionarla, noindexarla o eliminarla con criterio.
```

La persona nueva no necesita saberlo todo desde el primer dia.

Pero si debe aprender a no improvisar.

```text
SEO profesional no es meter keywords.
Es construir un sistema donde negocio, usuario, contenido, tecnica, autoridad y
medicion trabajan juntos.
```
