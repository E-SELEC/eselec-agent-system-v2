## Idea central

Google Business Profile, o GBP, es uno de los activos principales del SEO local.

No es una red social. No es solo una ficha con telefono.

```text
Para negocios fisicos o de servicio local, Google Business Profile puede ser
tan importante como la web.
```

GBP se optimiza para aumentar:

- relevancia local;
- confianza;
- interaccion real;
- visibilidad en Google Search;
- visibilidad en Google Maps;
- Local Pack;
- llamadas;
- rutas;
- clics a web;
- reseñas.

---

## Lugar en el sistema SEO

GBP conecta con:

1. SEO local: Google Maps, Local Pack, busquedas cerca de mi, ciudad/zona,
   llamadas, rutas, web y reseñas.
2. Reputacion: opiniones, respuestas, valoracion media, confianza y prominencia.
3. SEO On-Page: landing local, NAP consistente, servicios/productos y semantica
   local.
4. CRO local: llamadas, WhatsApp, reservas, rutas y contacto.
5. Medicion: rendimiento del perfil, interacciones, llamadas, clics, direcciones
   y busquedas.

Google basa resultados locales principalmente en:

```text
relevancia + distancia + prominencia
```

## Web nueva / negocio nuevo vs negocio existente

### Nuevo negocio local

Flujo:

```text
1. Confirmar elegibilidad
2. Crear o reclamar perfil
3. Verificar
4. Configurar nombre real
5. Elegir categoria principal
6. Añadir categorias secundarias
7. Añadir NAP
8. Añadir web correcta
9. Añadir horarios
10. Añadir servicios/productos
11. Subir fotos reales
12. Crear landing local
13. Conectar medicion
14. Empezar rutina de reseñas
```

### Negocio existente

Flujo:

```text
1. Auditar perfil actual
2. Confirmar propiedad y accesos
3. Revisar categoria principal
4. Revisar categorias secundarias
5. Revisar NAP
6. Revisar URL enlazada
7. Revisar servicios/productos
8. Revisar fotos
9. Revisar reseñas y respuestas
10. Revisar preguntas/respuestas
11. Revisar duplicados
12. Revisar rendimiento
13. Cruzar con web/GSC/GA4
14. Priorizar mejoras
```

## Elegibilidad y accesos

Antes de optimizar, confirmar:

- el negocio existe;
- cumple politicas de Google;
- tiene contacto real;
- tiene direccion o area de servicio valida;
- no hay perfil duplicado;
- Rodrigo/E-SELEC tiene acceso correcto.

Roles:

- propietario principal;
- propietario;
- administrador.

Regla:

```text
No se cambia un GBP si no esta claro quien es propietario y quien puede aprobar
cambios sensibles.
```

---

## Nombre, categorias y NAP

Nombre:

```text
Debe ser el nombre real del negocio.
```

No rellenar con keywords si no forman parte del nombre real.

Categoria principal:

```text
Es una de las decisiones mas importantes del perfil.
```

Proceso:

```text
1. Buscar competidores que posicionan bien.
2. Revisar que categoria usan.
3. Elegir la mas cercana al servicio principal real.
4. No elegir por deseo, sino por realidad del negocio.
5. Revisar si Google permite categoria exacta.
```

NAP:

```text
Name, Address, Phone.
```

Debe ser consistente entre GBP, web, citaciones, redes y directorios.

## Servicios, productos, fotos y publicaciones

Servicios:

- deben ser reales;
- deben coincidir con lo que vende el negocio;
- deben alinearse con landing local;
- deben ser claros para usuario y Google.

Productos:

- utiles para negocios locales con catalogo;
- deben tener nombre, descripcion, imagen y precio si aplica;
- deben representar oferta real.

Fotos/videos:

- fachada;
- interior;
- equipo;
- productos;
- trabajos reales;
- antes/despues si aplica;
- contexto local.

Publicaciones:

- novedades;
- ofertas reales;
- eventos;
- servicios destacados;
- contenido local.

Regla:

```text
Fotos reales y actividad sostenida construyen confianza local.
```

## Reseñas y reputacion

Las reseñas afectan confianza, conversion y prominencia.

Revisar:

- cantidad;
- frecuencia;
- calidad;
- texto usado por clientes;
- valoracion media;
- respuesta del negocio;
- reseñas negativas;
- patrones repetidos.

Proceso:

```text
1. Revisar reseñas nuevas.
2. Responder con tono profesional.
3. Detectar temas frecuentes.
4. Pedir reseñas de forma etica.
5. No inventar ni comprar reseñas.
6. Usar aprendizajes en web/contenido.
```

Regla:

```text
No se responde reseña solo para cerrar tarea; se responde para construir confianza.
```

## Preguntas, mensajes y rendimiento

Preguntas y respuestas:

- revisar preguntas reales;
- responder de forma clara;
- detectar dudas para FAQs de la web;
- mantener coherencia con servicios.

Mensajes/chat:

- activar solo si el negocio puede responder;
- medir calidad de respuesta;
- evitar prometer disponibilidad si no existe.

Rendimiento del perfil:

- llamadas;
- clics a web;
- solicitudes de ruta;
- interacciones;
- busquedas;
- dispositivos;
- tendencias.

## Conexion GBP + web + GSC + GA4

GBP no trabaja aislado.

Web:

- landing local correcta;
- NAP visible;
- contenido local real;
- servicios alineados;
- schema LocalBusiness si aplica;
- enlace desde GBP a URL correcta.

GSC:

- consultas locales;
- landings locales;
- rendimiento organico web.

GA4:

- sesiones desde GBP si se etiqueta;
- clics en telefono/WhatsApp;
- formularios;
- comportamiento en landing.

Regla:

```text
GBP muestra interaccion local en el perfil.
GSC muestra rendimiento organico de landings.
GA4 muestra comportamiento dentro de la web.
```

## Duplicados y suspensiones

Duplicados:

```text
1. Buscar nombre en Google Maps.
2. Buscar variantes.
3. Buscar direccion.
4. Buscar telefono.
5. Confirmar duplicados.
6. Reclamar, fusionar o reportar segun caso.
```

Suspensiones:

```text
1. No improvisar.
2. Revisar motivo.
3. Revisar politicas.
4. Reunir documentacion.
5. Corregir inconsistencias.
6. Apelar si corresponde.
```

Riesgo:

```text
Un cambio mal hecho en GBP puede afectar visibilidad local y confianza.
```

## Regla final para relevo

Google Business Profile no se optimiza para llenar campos.

```text
Se optimiza para aumentar relevancia local, confianza e interaccion real.
```

Cada trabajo en GBP debe poder responder:

```text
Que negocio estoy optimizando.
Que categoria principal tiene.
Que intencion local debe captar.
Que servicios reales ofrece.
Que landing local lo respalda.
Que reseñas tiene.
Que fotos necesita.
Que interacciones genera.
Que datos estan inconsistentes.
Que accion toca ejecutar.
Como se medira el resultado.
```

Si una persona dice "ya optimice la ficha", todavia no esta trabajando con
criterio.
