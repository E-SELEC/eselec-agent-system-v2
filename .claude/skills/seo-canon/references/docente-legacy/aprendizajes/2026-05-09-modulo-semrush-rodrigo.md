## Idea central

Este modulo no ensena botones.

Ensena al equipo a usar SEMrush como herramienta de:

```text
diagnostico -> investigacion -> priorizacion -> seguimiento
```

SEMrush debe ayudar a responder cuatro preguntas:

```text
1. Donde estamos ahora?
2. Que oportunidades existen?
3. Que errores estan frenando la web?
4. Que acciones debemos priorizar?
```

Regla base:

```text
SEMrush detecta.
El SEO interpreta.
El negocio prioriza.
El equipo ejecuta.
Los datos validan.
```

## 1. Que es SEMrush dentro del trabajo SEO

SEMrush es una suite SEO para analizar dominios, investigar keywords, auditar
problemas tecnicos, revisar backlinks, estudiar competidores, controlar
posiciones, preparar contenidos y crear reportes.

Herramientas mencionadas en el modulo:

```text
Domain Overview
Organic Rankings / Organic Research
Top Pages
Keyword Gap
Backlink Gap
Keyword Overview
Keyword Magic Tool
Keyword Strategy Builder
Site Audit
On Page SEO Checker
Organic Traffic Insights
SEO Writing Assistant
My Reports
```

Principio:

```text
SEMrush no decide la estrategia.
SEMrush entrega datos, senales y oportunidades.
El criterio SEO decide que hacer con esos datos.
```

Ejemplo:

```text
SEMrush puede decir:
Esta keyword tiene volumen y baja dificultad.

El SEO debe preguntar:
Tiene intencion comercial?
Tiene sentido para el negocio?
Ya tenemos una URL para esa intencion?
Podemos competir?
Convierte?
```

## 2. SEMrush en web nueva vs web existente

### Web nueva

SEMrush se usa para:

```text
investigar mercado,
detectar competidores organicos,
encontrar keywords,
agrupar intenciones,
disenar arquitectura SEO,
crear mapa keyword -> URL,
preparar contenidos,
revisar tecnica antes de publicar,
configurar seguimiento.
```

Idea:

```text
En web nueva, SEMrush ayuda a planificar antes de construir.
```

### Web existente

SEMrush se usa para:

```text
diagnosticar trafico estimado y visibilidad,
detectar keywords actuales,
encontrar paginas con oportunidad,
auditar errores tecnicos,
detectar canibalizacion,
mejorar URLs existentes,
revisar backlinks,
comparar contra competidores,
hacer seguimiento de posiciones,
crear reportes.
```

Idea:

```text
En web existente, SEMrush ayuda a medir, proteger, corregir y escalar.
```

## 5. Domain Overview

**Para que sirve**

Dar una primera fotografia del dominio: trafico organico estimado, trafico de
pago, backlinks, autoridad, competidores y visibilidad general.

**Web nueva**

Se usa para analizar competidores.

**Web existente**

Se usa para revisar estado general del dominio.

**Que mirar**

```text
Organic Traffic
Organic Keywords
Paid Traffic
Backlinks
Authority Score
AI Visibility, si aparece
Top Organic Keywords
Competidores organicos
Paginas principales
Tendencia de trafico
```

**Reglas**

```text
Organic Traffic es estimacion, no trafico real.
Authority Score es metrica de SEMrush, no metrica de Google.
Authority Score sirve para comparar, no como verdad absoluta.
```

Entregable:

```text
Dominio | Trafico organico estimado | Keywords | Backlinks | Authority Score | Fortaleza | Debilidad
```

Errores:

```text
Creer que trafico estimado es trafico real.
Obsesionarse con Authority Score.
Comparar negocios que no compiten por las mismas keywords.
Comparar webs nacionales con negocios locales pequenos.
Tomar decisiones solo desde Domain Overview.
```

## 6. Organic Research / Organic Rankings

**Para que sirve**

Detectar que keywords posiciona un dominio, con que URLs, en que posiciones y
que cambios hay.

**Web nueva**

Sirve para ver que paginas y contenidos funcionan en competidores.

**Web existente**

Sirve para ver keywords actuales, oportunidades, perdidas y URLs con potencial.

Filtros utiles:

```text
Posicion 4-10: mejorar para top 3.
Posicion 11-20: quick wins.
Posicion 21-50: oportunidad con mas trabajo.
Posicion perdida: revisar caida.
```

Interpretacion:

```text
Keyword en posicion 8 -> revisar la URL que ya posiciona, no crear otra.
Keyword en posicion 35 -> optimizacion profunda o URL nueva si no responde a intencion.
Keyword con URL equivocada -> posible canibalizacion o mala arquitectura.
```

Entregable:

```text
Listado de keywords actuales:
Quick wins:
Keywords perdidas:
URLs con oportunidad:
URLs mal alineadas:
Acciones recomendadas:
```

## 8. Keyword Magic Tool

**Para que sirve**

Ampliar investigacion desde una keyword semilla.

Proceso:

```text
1. Introducir keyword semilla.
2. Seleccionar pais.
3. Revisar grupos.
4. Filtrar por intencion.
5. Filtrar por dificultad.
6. Seleccionar keywords utiles.
```

Criterios para seleccionar:

```text
relevancia para el negocio,
intencion clara,
URL asignable,
volumen razonable,
dificultad asumible,
potencial de conversion.
```

Error:

```text
Exportar 2.000 keywords y no clasificarlas.
Eso no es keyword research. Eso es acumulacion de datos.
```

## 9. Keyword Gap

**Para que sirve**

Comparar keywords propias contra competidores para detectar oportunidades.

Filtros:

```text
Missing: competidor posiciona, nosotros no.
Weak: nosotros posicionamos peor.
Strong: nosotros posicionamos mejor.
Shared: todos comparten keyword.
Untapped: competidores rankean y nosotros no.
```

Proceso:

```text
1. Elegir competidores organicos reales.
2. Introducir dominio propio y 3-4 competidores.
3. Seleccionar Organic Keywords.
4. Revisar Missing y Weak.
5. Filtrar por intencion.
6. Decidir accion.
```

Decisiones:

```text
Si hay URL -> optimizar.
Si no hay URL -> crear pagina.
Si es informacional -> articulo.
Si es comercial -> landing.
Si es categoria -> categoria.
Si canibaliza -> no crear; revisar arquitectura.
```

Error:

```text
Copiar todas las keywords del competidor.
No se replica una web. Se extraen oportunidades relevantes.
```

## 10. Keyword Strategy Builder

**Para que sirve**

Convertir keywords en temas, paginas pilar y subpaginas.

Principio:

```text
Keyword research no termina en una lista.
Debe convertirse en arquitectura.
```

Validacion manual obligatoria:

```text
Esta pagina tiene intencion propia?
Puede posicionar sola?
Tiene valor comercial?
No canibaliza otra URL?
Tenemos capacidad de crear buen contenido?
```

Error:

```text
Pensar que cada keyword sugerida necesita una pagina propia.
```

## 11. Site Audit

**Para que sirve**

Rastrear la web y detectar problemas tecnicos y on-page.

Configuracion:

```text
alcance: dominio, subdominio o subcarpeta;
user-agent: Googlebot mobile si aplica;
limite de paginas: segun tamano;
frecuencia: semanal o mensual;
JavaScript rendering: activar si la web depende de JS.
```

Bloques principales:

```text
Crawlability
Indexability
HTTPS
Performance
Internal Linking
Markup
Core Web Vitals
International SEO, si aplica
```

Prioridad:

```text
Primero: errores que impiden rastreo/indexacion.
Segundo: errores en paginas con trafico o negocio.
Tercero: errores masivos de plantilla.
Cuarto: mejoras menores.
```

```text
Enviar el PDF de Site Audit sin interpretacion.
El valor esta en traducir errores en acciones claras.
```

## 12. On Page SEO Checker

**Para que sirve**

Dar ideas para mejorar paginas especificas, basadas en competidores top 10 y
keywords objetivo.

Casos ideales:

```text
URL en posicion 4-15.
Pagina con muchas impresiones y bajo CTR.
Pagina comercial importante.
Pagina con trafico pero baja conversion.
URL que necesita mejorar semantica.
```

Categorias de ideas:

```text
Strategy
SERP Features
Content
Semantic
Backlinks
Technical
User Experience
```

Regla:

```text
On Page SEO Checker orienta.
No sustituye analisis humano.
```

Error:

```text
Aplicar todas las ideas sin pensar.
```

## 13. Position Tracking

**Para que sirve**

Monitorizar rankings para keywords especificas por ubicacion, dispositivo y
buscador.

Regla:

```text
No se usa para ver posiciones por curiosidad.
Se usa para medir si las acciones funcionan.
```

Configurar:

```text
dominio,
ubicacion,
dispositivo,
keywords agrupadas por etiquetas,
competidores.
```

Interpretacion:

```text
Subio posicion -> revisar clics y conversiones.
Bajo posicion -> mirar tendencia semanal/mensual, no panico diario.
Posicion buena sin trafico -> revisar volumen, CTR o SERP.
Trafico sin posicion top -> puede venir de long-tail no rastreada.
```

Error:

```text
Medir 500 keywords sin agruparlas.
```

## 14. Cannibalization Report

**Para que sirve**

Detectar cuando varias paginas rankean para la misma keyword dentro del top 100.

Regla:

```text
No toda coincidencia de keyword es canibalizacion.
Lo importante es la intencion.
```

Decisiones:

```text
Misma intencion -> fusionar, redirigir o diferenciar.
Informacional + comercial -> pueden coexistir si se enlazan bien.
URL equivocada posiciona -> reforzar URL correcta.
Ambas tienen valor -> no eliminar sin revisar trafico, backlinks y conversiones.
```

## 17. Backlink Analytics / Backlink Gap / Backlink Audit

**Para que sirven**

Trabajar autoridad externa.

```text
Backlink Analytics -> revisar backlinks propios o de competidores.
Backlink Gap -> detectar sitios que enlazan a competidores pero no a nosotros.
Backlink Audit -> revisar salud y posibles riesgos del perfil de enlaces.
```

Prioridad:

```text
Alta: sitios relevantes, autoridad razonable, trafico real, contexto sectorial.
Media: directorios buenos, medios locales, asociaciones.
Baja: sitios genericos, sin trafico, sin relacion.
Evitar: PBNs, directorios basura, anchors manipulados, enlaces masivos.
```

Error:

```text
Buscar enlaces por cantidad, no por relevancia.
```

## 18. Organic Traffic Insights

**Para que sirve**

Cruzar SEMrush con Google Analytics y Google Search Console para analizar
landing pages organicas.

Uso principal:

```text
web existente.
```

Interpretacion:

```text
Pagina con trafico + conversiones -> proteger y reforzar.
Pagina con trafico + sin conversiones -> trabajar CRO.
Pagina con impresiones + pocos clics -> trabajar title/meta.
Pagina con keywords buenas + posicion media baja -> trabajar on-page/enlaces internos.
```

## 22. Como debe pensar el equipo dentro de SEMrush

La persona nueva debe memorizar:

```text
SEMrush detecta.
El SEO interpreta.
El negocio prioriza.
El equipo ejecuta.
Los datos validan.
```

Ejemplo:

```text
SEMrush dice:
Keyword: comprar pasta italiana online
Volumen: medio
KD: medio
Competidor rankea top 3
Nosotros no rankeamos

El SEO interpreta:
Vendemos pasta italiana online?
Tenemos categoria?
La URL existe?
La intencion es ecommerce?
Podemos competir?
Que hace mejor el competidor?
Necesitamos crear pagina o mejorar una existente?

Accion:
Crear/optimizar /pasta-italiana/
Anadir productos.
Anadir texto de categoria.
Anadir FAQs.
Anadir enlaces internos.
Anadir schema Product/Breadcrumb.
Medir rankings.
```

## 23. Errores comunes al usar SEMrush

```text
1. Creer que mas volumen siempre es mejor.
2. Crear paginas para cada keyword parecida.
3. Copiar competidores sin entender intencion.
4. Obsesionarse con Authority Score.
5. Tratar trafico estimado como trafico real.
6. Corregir todos los errores tecnicos sin priorizar.
7. Aplicar todas las ideas de On Page SEO Checker sin criterio.
8. No cruzar SEMrush con Search Console y GA4.
9. No diferenciar web nueva de web existente.
10. No convertir datos en tareas ejecutables.
```

## 26. Resumen final para la generacion de relevo

```text
SEMrush no se usa para llenar informes.
SEMrush se usa para encontrar decisiones.
```

La persona que use SEMrush debe terminar cada sesion con respuestas claras:

```text
Que encontramos?
Por que importa?
Que URL afecta?
Que accion se debe hacer?
Quien la hara?
Cuando se revisara?
Como sabremos si funciono?
```

Regla mas importante:

```text
Nunca salgas de SEMrush con solo datos.
Sal con tareas priorizadas.
```
