## Aprendizaje 1 - El Lider SEO no debe ejecutar, debe orquestar

El archivo `leader-seo.md` define al Lider SEO como coordinador.

Regla aprendida:

```text
El Lider SEO no hace todo.
Decide que subagente activa, en que orden y con que dependencia.
```

Dependencias principales:

```text
Si no hay web funcional:
activar WEB antes de SEO.

Si hay problemas tecnicos criticos:
SEO Tecnico va primero.

Si la web es nueva:
SEO Web valida arquitectura antes de tecnico, organico o local.

Si se quiere SEO para LLMs:
SEO Organico debe haber trabajado contenido base antes.
```

Esto coincide con el marco de Rodrigo:

```text
SEO no es hacer cosas sueltas.
Es un sistema donde cada accion afecta a otra.
```

## Aprendizaje 2 - La regla SEMrush + GSC ya esta instalada

`leader-seo.md`, `seo-organico.md` y `seo-local.md` ya contienen una regla importante:

```text
SEMrush estima, compara y detecta oportunidades.
GSC valida datos reales de Google.
```

El Docente debe conservar esta logica.

Pero debe reforzarla con lo aprendido en los modulos:

```text

GA4 no muestra queries completas.
La decision sale del cruce, no de una herramienta sola.
```

## Aprendizaje 4 - SEO Organico ya trabaja intencion, pero hay que afinar canibalizacion

`seo-organico.md` tiene bases correctas:

```text
intencion antes de escribir
keywords comerciales y transaccionales primero
maximo 20-30 keywords
clusters tematicos
enlazado interno
contenido listo para WordPress
quick wins por posiciones 11-20
```

El Docente debe ajustar un matiz importante.

La frase operativa:

```text
Una pagina = una keyword principal.
```

Debe ensenarse asi:

```text
Una URL = una intencion principal.
Una URL puede trabajar una keyword principal y secundarias
si todas responden la misma intencion.
```

Esto evita que un agente cree paginas innecesarias por cada variacion de keyword.

## Aprendizaje 6 - SEO LLMs necesita actualizacion conceptual futura

`seo-llms.md` ya introduce AEO/GEO, preguntas conversacionales, consulta directa de LLMs, schema y autoridad externa.

Es una buena base.

Pero el Docente debe ensenar una version mas precisa tras el modulo de SEO para IA externo:

```text
No todos los sistemas IA funcionan igual.
Algunos usan buscadores.
Algunos rastrean.
Algunos recuperan contenido por accion del usuario.
Algunos entrenan modelos.
Algunos citan fuentes externas.
```

Por tanto, la frase:

```text
Los LLMs no indexan como Google.
```

debe explicarse con matiz, no como regla absoluta.

El Docente debe ensenar:

```text
SEO para IA externo exige distinguir Google AI Overviews,
AI Mode, ChatGPT Search, Perplexity, Copilot, Claude,
bots de entrenamiento, bots de busqueda y bots de usuario.
```

No se debe tocar `robots.txt`, `llms.txt` o permisos de bots sin criterio senior.

## Aprendizaje 8 - Las referencias SEMrush son utiles, pero no son autoridad oficial

`semrush-workflows.md` y `seo-onpage-semrush.md` son bases utiles para ejecutar.

Contienen:

```text
workflows por herramienta
insights por agente
criterios on-page
checklists
umbrales
metodos de auditoria
```

El Docente debe clasificarlas correctamente:

```text
Semrush = metodologia de herramienta y apoyo operativo.
Google = fuente oficial sobre rastreo, indexacion, Search, GA4 y GSC.
Rodrigo = marco interno de decision y ejecucion.
```

Las metricas y umbrales de Semrush o Neil Patel pueden orientar, pero no deben convertirse en dogma sin verificacion.
