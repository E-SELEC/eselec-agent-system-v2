# Checklist de revision - seo-audit

Antes de entregar, verifica:

## Datos

- [ ] Hay dominio o URL.
- [ ] Declare alcance.
- [ ] Declare nivel de datos.
- [ ] Declare fuentes usadas.
- [ ] Marque GSC faltante si no existe.
- [ ] Marque SEMrush faltante si no existe.
- [ ] No afirme metricas que no consulte.

## SEO

- [ ] Revise crawlability/indexacion antes de contenido.
- [ ] No propuse contenido si hay bloqueo tecnico critico.
- [ ] Cada hallazgo importante tiene evidencia.
- [ ] Top 3 problemas estan priorizados.
- [ ] Separé tecnico, on-page/contenido, autoridad/local/AI si aplica.
- [ ] Inclui matriz impacto/esfuerzo.
- [ ] Deje plan de accion en orden.

## Schema

- [ ] No concluí ausencia de schema usando solo HTML estatico.
- [ ] Si no renderice, marque schema como no verificado.

## Riesgo

- [ ] No propuse tocar produccion sin activos criticos.
- [ ] Redirecciones, DNS, WordPress, WooCommerce o GBP quedan como Orden de Cambio si se ejecutan.
- [ ] Si faltan datos, el output dice parcial.

## Cierre

- [ ] Hay siguiente paso concreto.
- [ ] Si se guardo output, se actualizo manifest.
- [ ] Si hubo cambios de archivos, se actualizo registro/log.
