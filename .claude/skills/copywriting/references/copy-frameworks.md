# Frameworks de copy

Usar como punto de partida, no como molde rigido.

## Hero

```text
Headline: [resultado claro] para [audiencia] sin/con [diferenciador]
Subheadline: [como se logra] + [por que importa]
CTA: [accion] + [lo que obtiene]
Prueba: [dato/testimonio/logos si existe]
```

## Landing

1. Hero.
2. Problema reconocible.
3. Solucion.
4. Beneficios principales.
5. Como funciona.
6. Prueba/confianza.
7. FAQ/objeciones.
8. CTA final.

## Pagina de servicio

1. Que problema resuelve.
2. Para quien es.
3. Que incluye.
4. Como trabaja E-SELEC o el cliente.
5. Resultados esperables sin prometer.
6. Prueba o experiencia.
7. CTA.

## Headline formulas

- [Resultado] sin [dolor]
- [Servicio/categoria] para [audiencia]
- Convierte [situacion actual] en [resultado]
- Deja de [dolor]. Empieza a [beneficio]
- La forma [diferenciador] de conseguir [resultado]
- [Pregunta que refleja dolor real]

## CTA formulas

- Solicitar [resultado]
- Ver [recurso/oferta]
- Reservar [tipo de llamada]
- Pedir presupuesto
- Descargar [recurso]
- Revisar mi [activo]

Evita CTAs vagos: "Enviar", "Mas informacion", "Click aqui".

## Beneficio bien escrito

```text
[Resultado] porque [mecanismo].
```

Ejemplo:

```text
Recibe mas solicitudes cualificadas porque cada pagina responde a una intencion de busqueda concreta.
```
