# Knowledge Inbox

Usar solo como entrada temporal para informacion pendiente de clasificar.

Reglas:

- No guardar secretos ni tokens.
- No dejar archivos aqui como estado final.
- Cada item debe terminar en uno de estos estados:
  - movido a `knowledge/[fuente]/`;
  - convertido en protocolo, skill, agente, command, memoria o evidencia;
  - archivado en `legacy/`;
  - descartado.

Antes de cerrar una tarea, el inbox debe quedar explicado en `registries/registro-fuentes.md` o en `registries/registro-artefactos.md`.
