# Brief de conversacion fuente SEM / Paid Ads

Fecha: 2026-05-20
Estado: preparado, no ejecutado
Alcance: SEM / Paid Ads general para E-SELEC v2

## Decision

SEM / Paid Ads sera la primera area no SEO en pasar por el metodo de conversacion fuente.

No se crea canon todavia. Primero se construye un manual operativo profundo, usando fuentes oficiales de cada plataforma y el mismo patron que funciono en SEO:

1. canon claro,
2. separacion entre informacion y criterio,
3. foco por area,
4. reglas mentales fuertes,
5. ejecucion explicada para una persona nueva,
6. fuentes primarias verificables,
7. nada de ejemplos de clientes reales dentro de material general.

## Por que no crear el canon directo

El canon SEO funciono porque no nacio como resumen rapido. Nacio de una conversacion formativa exigente que obligo a explicar criterio, procesos, herramientas, errores y fuentes.

Crear el canon SEM directamente desde la skill actual `paid-ads` seria mas rapido, pero mas debil. La skill ya sirve para operar; lo que falta es la capa profunda de criterio.

## Objetivo de la conversacion

Crear un manual operativo de SEM / Paid Ads para formar criterio en agentes y personas nuevas de E-SELEC.

El resultado debe permitir que alguien entienda:

- que mirar antes de tocar una cuenta publicitaria,
- que decision tomar segun objetivo, presupuesto, tracking, audiencia y oferta,
- que no tocar sin supervision,
- como distinguir optimizacion real de cambios impulsivos,
- como documentar decisiones de campana,
- como convertir informacion de plataforma en criterio operativo.

## Alcance inicial

Orden recomendado:

1. Google Ads: base estructural de SEM, busqueda, objetivos, conversiones, presupuesto y puja.
2. Meta Ads: performance social, estructura campana/ad set/ad, pixel, Conversions API, creatividades y aprendizaje.
3. LinkedIn Ads: B2B, jerarquia de Campaign Manager, targeting, presupuesto y medicion.
4. TikTok Ads: creatividad, estructura campaign/ad group/ad, objetivos y medicion.

No mezclar todas las plataformas en un solo bloque. Cada plataforma debe tener modulos propios.

## Estructura obligatoria por modulo

Cada modulo de la conversacion debe responder:

1. Que es y para que sirve.
2. Que problema resuelve en una cuenta real.
3. Que debe revisar primero alguien nuevo.
4. Que fuente oficial sostiene cada afirmacion importante.
5. Que decision debe poder tomar despues de leerlo.
6. Cual es la regla dura del modulo.
7. Cual es el proceso paso a paso.
8. Que error comete normalmente un junior.
9. Que no debe tocar sin supervision.
10. Que checklist debe pasar antes de cerrar.

## Reglas duras del manual

- No inventar datos no proporcionados.
- No usar ejemplos de clientes reales en material general.
- Si se usa un ejemplo, marcarlo como ficticio.
- Cada recomendacion tecnica debe apuntar a fuente oficial o declarar que es criterio interno.
- Separar informacion, criterio e inferencia.
- No mezclar plataformas como si fueran equivalentes.
- No recomendar cambios de presupuesto, puja, tracking, pixel, conversiones o publicacion sin supervision.
- El manual debe ser formativo: debe ensenar a pensar, no solo listar botones.

## Prompt base para abrir la conversacion

```text
Imagina que eres un especialista senior en SEM / Paid Ads con experiencia real gestionando cuentas de Google Ads, Meta Ads, LinkedIn Ads y TikTok Ads.

Tu rol es construir un manual operativo profundo para formar a una persona nueva dentro de una agencia. Esta persona no sabe ejecutar SEM todavia, pero sera responsable de aprender a revisar, diagnosticar, proponer y operar cuentas publicitarias con criterio.

No quiero un glosario ni un resumen. Quiero un manual de uso formativo, ordenado, coherente, detallado y aplicable.

Reglas:
- No inventes datos.
- Si usas ejemplos ficticios, dilo.
- Usa fuentes oficiales de cada plataforma como base.
- Manten los links dentro del contenido.
- Separa informacion, criterio e inferencia.
- Explica como decidir, no solo que hacer.
- No uses ejemplos de clientes reales.
- No rellenes. Cada palabra debe aportar.

Primero trabajaremos Google Ads como base. Antes de escribir el manual, define la estructura completa de modulos que debe tener una persona nueva para entender Google Ads de forma profesional: objetivos, tipos de campana, conversiones, presupuesto, puja, keywords, audiencias, anuncios, extensiones/assets, landing page, medicion, optimizacion, errores comunes y limites de accion.

Para cada modulo indica:
1. que aprendera,
2. que decisiones podra tomar,
3. que fuentes oficiales se usaran,
4. que no debe tocar sin supervision.
```

## Prompt de profundizacion por modulo

```text
Ahora desarrolla el modulo [NOMBRE DEL MODULO] con profundidad profesional.

Verifica que la informacion sea correcta segun fuentes oficiales de [PLATAFORMA].

Explica a alguien sin conocimiento previo:
- que es,
- para que sirve,
- como se configura o revisa,
- que senales debe mirar,
- que errores debe evitar,
- cuando debe escalar a alguien senior,
- que checklist debe completar antes de cerrar.

No tienes problema en extenderte. La meta es formar criterio operativo real, no producir un resumen. Manten el tono profesional, claro, ordenado, coherente, detallado y formativo, sin relleno ni canibalizacion.

Incluye links oficiales dentro del contenido y explica por que cada fuente importa.
```

## Prompt de conversion a manual reutilizable

```text
Convierte lo trabajado en un manual operativo reutilizable para E-SELEC.

Condiciones:
- Manten la informacion en el mismo orden logico de aprendizaje.
- No resumas si se pierde criterio.
- No reescribas para hacerlo mas corto.
- Separa informacion, criterio, proceso, checklist y limites.
- El resultado debe poder convertirse luego en canon, skill o referencia sin contaminarse con casos reales.
```

## Criterio de cierre

Un modulo esta listo solo si contiene:

- fuentes oficiales,
- explicacion para principiante,
- criterio de decision,
- proceso paso a paso,
- errores comunes,
- limites de accion,
- checklist de cierre.

Si falta una de esas partes, el modulo no se considera manual operativo.

## Acuerdo con Claude

Claude reviso el enfoque y recomendo siete campos obligatorios para evitar falsos manuales:

- alcance declarado,
- perfil del lector,
- pregunta de salida por modulo,
- fuente primaria por plataforma,
- estructura repetible,
- separacion cliente/doctrina,
- senal de parada.

La recomendacion central fue: si la conversacion empieza en "que es Google Ads", termina como glosario; si empieza con estructura, fuente y criterio de cierre, puede convertirse en manual.

## Siguiente accion

Abrir la conversacion fuente SEM / Paid Ads usando este brief.

No mover nada a `seo-canon`, no mezclar con clientes y no crear canon SEM hasta tener el manual operativo completo.
